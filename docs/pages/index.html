OpenSPODES

Portable C11 DLMS/COSEM Protocol Stack for Smart Metering

v1.2.0 GPL-3.0 C11

About

OpenSPODES is a portable, zero-heap-allocation C11 implementation of the IEC 62056 DLMS/COSEM protocol stack. Designed for embedded systems, smart meters, and concentrators with MCU-friendly architecture.

Features

DLMS/COSEM Full Stack

BER/AXDR codec, HDLC session + wrapper framing, ACSE, xDLMS GET/SET/ACTION

42 COSEM IC Classes

Data, Register, ProfileGeneric, Clock, AssociationLN, SecuritySetup, and more

Security Suites

AES-GCM-128, GOST Kuznyechik, ECDH-ECDSA, Streebog — HLS mechanisms 0-10

Zero Heap Allocation

All buffers are static, no malloc/free — safe for bare-metal MCU targets

HAL Abstraction

Pluggable transport, crypto, timer, random — port to any MCU/RTOS

Thread-Safe

Optional HAL mutex for multi-threaded use (Linux, FreeRTOS, Zephyr)

Quick Start

# Build
cmake -S . -B build -DCMAKE_BUILD_TYPE=Debug
cmake --build build -j$(nproc)

# Run loopback demo
./build/openspodes_loopback_cli demo

# Run tests
ctest --test-dir build --output-on-failure

HLS Authentication

OpenSPODES supports all 11 HLS mechanisms (0-10) per IEC 62056-5-3 Green Book:

  • Mechanism 5 (GMAC) — AES-GCM authentication tag
  • Mechanism 8 (GOST-CMAC) — Kuznyechik CMAC
  • Mechanism 10 (GOST-Sig) — GOST 34.10 digital signature
